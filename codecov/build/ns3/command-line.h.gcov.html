      50             :  * \brief Parse command-line arguments
      51             :  *
      52             :  * Instances of this class can be used to parse command-line 
      53             :  * arguments.  Programs can register a general usage message with
      54             :  * CommandLine::Usage, and arguments with CommandLine::AddValue.
      55             :  * Argument variable types with input streamers (`operator>>`)
      56             :  * can be set directly; more complex argument parsing 
      57             :  * can be accomplished by providing a Callback.
      58             :  *
      59             :  * CommandLine also provides handlers for these standard arguments:
      60             :  * \verbatim
      61             :    --PrintGlobals:              Print the list of globals.
      62             :    --PrintGroups:               Print the list of groups.
      63             :    --PrintGroup=[group]:        Print all TypeIds of group.
      64             :    --PrintTypeIds:              Print all TypeIds.
      65             :    --PrintAttributes=[typeid]:  Print all attributes of typeid.
      66             :    --PrintHelp:                 Print this help message. \endverbatim
      67             :  * 
      68             :  * The more common \c --help is a synonym for \c --PrintHelp; an example
      69             :  * is given below.
      70             :  *
      71             :  * CommandLine can also handle non-option arguments
      72             :  * (often called simply "positional" parameters: arguments which don't begin
      73             :  * with "-" or "--").  These can be parsed directly in to variables,
      74             :  * by registering arguments with AddNonOption in the order expected.
      75             :  * Additional non-option arguments encountered will be captured as strings.
      76             :  *
      77             :  * Finally, CommandLine processes Attribute and GlobalValue arguments.
      78             :  * Default values for specific attributes can be set using a shorthand
      79             :  * argument name.
      80             :  *
      81             :  * In use, arguments are given in the form
      82             :  * \verbatim
      83             :    --arg=value --toggle first-non-option\endverbatim
      84             :  * Most arguments expect a value, as in the first form, \c --arg=value.
      85             :  * Toggles, corresponding to boolean arguments, can be given in any of
      86             :  * the forms
      87             :  * \verbatim
      88             :    --toggle1 --toggle2=1 --toggle3=t --toggle4=true \endverbatim
      89             :  * The first form changes the state of toggle1 from its default; 
      90             :  * all the rest set the corresponding boolean variable to true.
      91             :  * \c 0, \c f and \c false are accepted to set the variable to false.
      92             :  * Option arguments can appear in any order on the command line,
      93             :  * even intermixed with non-option arguments.
      94             :  * The order of non-option arguments is preserved.
      95             :  *
      96             :  * Option arguments can be repeated on the command line; the last value given
      97             :  * will be the final value used.  For example,
      98             :  * \verbatim
      99             :    --arg=one --toggle=f --arg=another --toggle \endverbatim
     100             :  * The variable set by \c --arg will end up with the value \c "another";
     101             :  * the boolean set by \c --toggle will end up as \c true.
     102             :  *
     103             :  * Because arguments can be repeated it can be hard to decipher what
     104             :  * value each variable ended up with, especially when using boolean toggles.
     105             :  * Suggested best practice is for scripts to report the values of all items
     106             :  * settable through CommandLine, as done by the example below.
     107             :  * 
     108             :  *
     109             :  * CommandLine can set the initial value of every attribute in the system
     110             :  * with the

     112             :  * \verbatim
     113             :    --Application::StartTime=3s \endverbatim

     115             :  * attribute for a simulation script.  For example, you might want
     116             :  * to make it easy to set the \c Application::StartTime using
     117             :  * the argument \c --start, and have its help string show as part

     121             :  *
     122             :  * CommandLine can also set the value of every GlobalValue
